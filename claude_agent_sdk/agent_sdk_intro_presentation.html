The Problem with Building Agents Today

Without Agent SDK

  • You implement the tool loop yourself
  • You build tool execution, retries, context management
  • You handle permissions, sandboxing, error recovery
  • Every project re-invents the same orchestration
# Client SDK — you own the loop

With Agent SDK

  • Claude handles tools autonomously
  • Built-in file I/O, shell, web, and search tools
  • Permission modes, hooks, sandboxing included

Custom Tools

Extend Claude with your own Python functions via in-process MCP

03

Defining Custom Tools

@tool decorator

from claude_agent_sdk import tool
from typing import Any

@tool("lookup_user",
     "Look up a user by email",
     {"email": str})
async def lookup_user(
    args: dict[str, Any]
) -> dict[str, Any]:
    user = await db.find(args["email"])
    return {
        "content": [{
            "type": "text",
            "text": f"User: {user.name}"
        }]
    }

Input schema supports simple types or full JSON Schema.

Wire it up

from claude_agent_sdk import (
    create_sdk_mcp_server,
    ClaudeSDKClient, ClaudeAgentOptions
)

server = create_sdk_mcp_server(
    name="my_tools",
    tools=[lookup_user]
)

options = ClaudeAgentOptions(
    mcp_servers={"tools": server},
    allowed_tools=[
        "mcp__tools__lookup_user"
    ]
)

async with ClaudeSDKClient(options) as c:
    await c.query("Find alice@co.com")
    async for msg in c.receive_response():
        print(msg)

Key insight: Custom tools run as in-process MCP servers — no separate process, no network hop. Tool name format: mcp__<server>__<tool>

Subagents

Parallel, context-isolated agents with specialized instructions

04

Defining & Using Subagents

from claude_agent_sdk import query, ClaudeAgentOptions, AgentDefinition

async for msg in query(
    prompt="Review auth module, then run tests",
    options=ClaudeAgentOptions(
        allowed_tools=["Read", "Grep", "Glob", "Task"],  # Task required for subagents
        agents={
            "code-reviewer": AgentDefinition(
                description="Expert code reviewer for security and quality.",
                prompt="Analyze code quality. Be thorough but concise.",
                tools=["Read", "Grep", "Glob"],   # read-only access
                model="sonnet"
            ),
            "test-runner": AgentDefinition(
                description="Runs and analyzes test suites.",
                prompt="Run tests and provide clear analysis of results.",
                tools=["Bash", "Read", "Grep"]  # can execute commands
            ),
        }
    )
):
    if hasattr(msg, "result"):
        print(msg.result)

Context Isolation

Each subagent has its own context window. Only relevant results flow back to the orchestrator.

Parallelization

Claude can spawn multiple subagents concurrently — e.g. review + test + lint simultaneously.

Tool Restrictions

Limit subagents to specific tools. Omit tools to inherit all. Subagents cannot spawn subagents.

Hooks

Intercept and control agent behavior at every execution point

05

Hook System

Available Hook Events

EventWhen
PreToolUseBefore tool executes (can block/modify)
PostToolUseAfter tool completes (audit/log)
UserPromptSubmitWhen user sends a prompt
StopAgent finishing execution
SubagentStopSubagent completes
PreCompactBefore context compaction

Callbacks receive (input_data, tool_use_id, context) and return a dict.

Example: Block Dangerous Commands

async def guard_bash(input_data, tool_use_id, ctx):
    cmd = input_data['tool_input'].get('command', '')
    if 'rm -rf /' in cmd:
        return {
            'hookSpecificOutput': {
                'hookEventName': 'PreToolUse',
                'permissionDecision': 'deny',
                'permissionDecisionReason':
                    'Blocked: destructive command'
            }
        }
    return {}

options = ClaudeAgentOptions(
    hooks={
        'PreToolUse': [
            HookMatcher(
                matcher='Bash',
                hooks=[guard_bash]
            )
        ]
    }
)

Hooks: Transform & Audit

Redirect Writes to Sandbox

async def sandbox_writes(data, tid, ctx):
    path = data['tool_input'].get('file_path', '')
    return {
        'hookSpecificOutput': {
            'hookEventName': 'PreToolUse',
            'permissionDecision': 'allow',
            'updatedInput': {
                **data['tool_input'],
                'file_path': f'/sandbox{path}'
            }
        }
    }

# Matcher: only Write and Edit tools
HookMatcher(
    matcher='Write|Edit',
    hooks=[sandbox_writes]
)

Audit Log Every Tool Call

from datetime import datetime

async def audit_log(data, tid, ctx):
    with open('audit.log', 'a') as f:
        f.write(
            f"{datetime.now().isoformat()} "
            f"tool={data['tool_name']} "
            f"id={tid}\n"
        )
    return {}

# No matcher → runs for ALL tools
HookMatcher(hooks=[audit_log])

Hook Decision Priority

deny
→
ask
→
allow

Any deny → blocked. Multiple hooks chain in array order.

Custom can_use_tool

from claude_agent_sdk.types import (
    PermissionResultAllow,
    PermissionResultDeny
)

async def authorize(
    tool_name: str,
    input_data: dict,
    context: ToolPermissionContext
) -> PermissionResult:
    # Block writes to system dirs
    path = input_data.get("file_path", "")
    if path.startswith("/system/"):
        return PermissionResultDeny(
            message="System dir blocked",
            interrupt=True
        )
    # Redirect config writes
    if "config" in path:
        return PermissionResultAllow(
            updated_input={
                **input_data,
                "file_path": f"./safe/{path}"
            }
        )
    return PermissionResultAllow()

Session Continuity

ClaudeSDKClient — Multi-Turn

async with ClaudeSDKClient(options) as c:
    # Turn 1
    await c.query("Read the auth module")
    async for msg in c.receive_response():
        print_text(msg)

    # Turn 2 — Claude remembers context
    await c.query("Now find all callers of it")
    async for msg in c.receive_response():
        print_text(msg)

    # Turn 3 — still same session
    await c.query("Refactor to use JWT")
    async for msg in c.receive_response():
        print_text(msg)

Resume with query()

session_id = None

# First query — capture session
async for msg in query(
    prompt="Read auth module",
    options=ClaudeAgentOptions(
        allowed_tools=["Read", "Glob"]
    )
):
    if hasattr(msg, 'session_id'):
        session_id = msg.session_id

# Second query — resume session
async for msg in query(
    prompt="Find all callers",
    options=ClaudeAgentOptions(
        resume=session_id
    )
):
    if hasattr(msg, "result"):
        print(msg.result)

Putting It Together

Production patterns and real-world agent design

08

Full Example: CI Code Review Agent

import asyncio
from claude_agent_sdk import (
    query, ClaudeAgentOptions, AgentDefinition,
    HookMatcher, ResultMessage
)

async def block_force_push(data, tid, ctx):
    cmd = data['tool_input'].get('command', '')
    if '--force' in cmd or 'push -f' in cmd:
        return {'hookSpecificOutput': {'hookEventName': 'PreToolUse',
                 'permissionDecision': 'deny', 'permissionDecisionReason': 'No force push in CI'}}
    return {}

async def main(pr_diff: str):
    async for msg in query(
        prompt=f"Review this PR diff and post findings:\n{pr_diff}",
        options=ClaudeAgentOptions(
            system_prompt="You are a senior code reviewer. Focus on bugs, security, and performance.",
            allowed_tools=["Read", "Grep", "Glob", "Bash", "Task"],
            permission_mode="bypassPermissions",   # CI — no interactive prompts
            max_turns=25,
            max_budget_usd=2.0,
            hooks={'PreToolUse': [HookMatcher(matcher='Bash', hooks=[block_force_push])]},
            agents={
                "security-scanner": AgentDefinition(
                    description="Scans code for OWASP vulnerabilities.",
                    prompt="Check for injection, XSS, auth issues.",
                    tools=["Read", "Grep", "Glob"], model="sonnet"
                )
            }
        )
    ):
        if isinstance(msg, ResultMessage):
            print(f"Review done. Cost: ${msg.total_cost_usd:.4f}")

Error Handling & ResultMessage

Structured Errors

from claude_agent_sdk import (
    CLINotFoundError,
    ProcessError,
    CLIConnectionError,
    CLIJSONDecodeError
)

try:
    async for msg in query(prompt="..."):
        print(msg)
except CLINotFoundError:
    print("Install Claude Code first")
except ProcessError as e:
    print(f"Exit code: {e.exit_code}")
    print(f"stderr: {e.stderr}")
